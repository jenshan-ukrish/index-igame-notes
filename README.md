# index-igame-notes

本仓库用于归档与整理多个独立 HTML 页面，作为个人或项目相关的页面笔记集合。仓库内容不针对任何特定域名或网站，仅作为静态页面的统一存放与版本管理用途。

## 项目简介

`index-igame-notes` 是一个以 HTML 页面为主要内容的归档仓库。每个页面通常以独立文件形式存在，内容涵盖各类记录、说明或实验性页面。仓库本身不提供动态服务，仅通过 GitHub Pages 或其他静态托管方式展示页面内容。

## 目录说明

```
.
├── README.md          # 本文件
├── index.html         # 仓库首页或导航页（如有）
├── pages/             # 存放主要 HTML 页面文件
│   ├── example1.html
│   └── example2.html
└── assets/            # 页面引用的资源文件（样式、脚本、图片等）
    ├── style.css
    └── script.js
```

- `pages/`：核心页面目录，所有独立 HTML 归档文件存放于此。
- `assets/`：页面所需的静态资源，按需组织。
- 根目录下的 `index.html` 可作为仓库入口页或页面索引。

## 页面归档说明

- 每个 HTML 页面视为一个独立的笔记或内容单元。
- 页面之间无强依赖关系，可单独访问。
- 归档内容以通用、自包含为原则，尽量减少外部依赖。
- 文件名应简洁且具描述性，便于识别与引用。

## 维护说明

- 新增页面：在 `pages/` 目录下创建新的 HTML 文件，如有资源文件请同步放入 `assets/`。
- 更新页面：直接修改对应 HTML 文件，保持版本历史可追溯。
- 删除页面：移除文件并清理相关资源引用。
- 建议在提交时附上简洁的 commit 信息，说明变更内容。

## 使用方式

1. 克隆本仓库到本地：
   ```bash
   git clone https://github.com/your-username/index-igame-notes.git
   ```
2. 直接在浏览器中打开 `pages/` 下的 HTML 文件即可查看。
3. 若启用 GitHub Pages，可访问对应 URL 浏览页面集合。
